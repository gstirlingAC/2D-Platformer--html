Input Map

Before we dive into scripting our player’s movement, jumping, and physics, we need to establish our inputs. Inputs define which buttons on your keyboard, mouse, or controller perform specific actions. Instead of listening for specific keys like the left arrow key or the spacebar, we will listen for actions like ‘jump’ or ‘move_left’. This approach makes our script more flexible and easier to manage.

Why Use Input Actions?

  • Abstracts the input method from the script logic.
  • Allows for multiple keys to trigger the same action.
  • Makes it easier to add or change input methods, such as adding a controller.

Setting Up Input Actions

  1. Navigate to the input map:
    • Go to Project in the top menu.
    • Select Project Settings.
    • Click on the Input Map tab.
  2. Create a new action:
    • Scroll down to Add New Action.
    • Name your action (e.g., move_left) and press Enter.
  3. Assign keys to the action:
    • Click the plus icon on the right.
    • Search for the desired key or click on ‘Listening for input’ and press the key.
    • Click OK to assign the key.
    • To add multiple keys, repeat the process.
  4. Repeat the process for other actions (e.g., move_right, jump).

Example of Input Actions

  • move_left: Left Arrow Key, A Key
  • move_right: Right Arrow Key, D Key
  • jump: Up Arrow Key, Spacebar, W Key

By setting up your inputs in this way, you can easily manage and update your input methods without changing your script logic. This separation makes your code cleaner and more maintainable.
